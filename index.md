# AI駆動開発プレイブック

## 概要

SIer組織（10〜30名チーム）向けのAI活用開発プレイブック集です。
開発者が **「いつ・何を・どうやって」** AIを活用するかを、工程別・ユースケース別にガイドします。

## 対象（初版）

| 項目 | スコープ |
|------|----------|
| 工程 | 実装 / 単体テスト |
| AIツール | GitHub Copilot（Agent Mode / Chat / Inline Completion） |
| 技術スタック | Java / Spring Boot |
| 案件タイプ | Webアプリ改修案件（ウォーターフォール） |

## サイト構成

| セクション | 内容 |
|-----------|------|
| [ダッシュボード](dashboard) | コンテンツの可視化・統計情報 |
| [ドキュメント](docs-index) | ガイドドキュメント（共通・実装・テスト・リサーチ・レビュー） |
| [参考資料](reference-index) | Copilot設定ファイル・エージェント・スキル定義 |

## AI活用の考え方

**AIはスキルの代替ではなく、増幅ツールである。**
AIの出力を読み解き、適切に判断・修正すること自体が高いスキルを要求される。理解していないコードはマージしないという原則はAI活用でも変わらない。

## 方針

- **小さく始めて改善** — まず1つのUCを試し、チームでふりかえって育てる
- **読まれるボリューム感** — 各UC 60〜100行。長すぎて読まれないドキュメントは作らない
- **教科書ではなく毎日開くガイド** — 理論より「今日の作業で使える」を優先
