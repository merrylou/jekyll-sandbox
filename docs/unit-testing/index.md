# 単体テスト工程プレイブック

## 工程概要

実装コードに対する単体テストを作成・実行する工程。JUnit5 + Mockito + Spring Boot Test を使用。

---

## AI活用の原則

1. **テスト設計の最終判断は人間** — テスト観点の網羅性・妥当性はAI任せにしない
2. **AI生成テストはカバレッジ計測と併用** — 生成されたテストが実際にどこを通っているか必ず計測する
3. **期待値（assert）は必ず人間が確認** — AIが生成した期待値が仕様と一致しているか目視で検証する
4. **テストデータに本番データ・個人情報を使わない** — AI生成のテストデータにも個人情報が含まれていないか確認する

## どのUCから始めるか

- **最も効果を実感しやすいのは UC01（テスト自動生成）** — テストコードの初稿をAIに任せることで、作成時間を大幅に短縮できる。生成後にassertの妥当性確認とカバレッジ計測を行うフローを習慣化する
