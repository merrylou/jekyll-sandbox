# UC01: 実装コードからのテスト自動生成

## 目的

実装済みのServiceクラス等に対し、JUnit5 + Mockitoの単体テストを一括生成する。テストコード作成の初期工数を大幅に削減する。

## 利用タイミング

対象クラスの実装が完了し、コードレビュー前にテスト作成へ着手するタイミング。

## Input / Output

| 区分   | 項目                         | 説明                                       |
| ------ | ---------------------------- | ------------------------------------------ |
| Input  | テスト対象のJavaソースコード | Serviceクラス等の実装済みファイル           |
| Input  | テスト方針                   | 正常系・異常系・境界値のどこまで網羅するか |
| Output | JUnit5テストクラス           | `*Test.java` ファイル                      |
| Output | テストケース一覧             | 生成したテストメソッドの一覧と観点         |

## 使い方

> **Copilotユーザー:** スラッシュコマンド `/ut-create-test` で実行できます。以下はコマンドの内容を理解・カスタマイズするための参考情報です。

### パターンA: Agent Mode（推奨）

```
{{テスト対象ファイルパス}} を読み取り、
以下の方針でJUnit5 + Mockitoの単体テストを生成してください。

【テスト方針】
- 正常系・異常系・境界値を網羅する
- @ExtendWith(MockitoExtension.class) を使用する
- テスト対象クラスの依存は @Mock で定義し、@InjectMocks でテスト対象を初期化する
- アサーションは AssertJ を使用する
- テストメソッド名は日本語で「正常系_〇〇の場合_〇〇を返す」形式にする

【参照すべき既存パターン】
- {{既存テストファイルパス}} のスタイルに合わせること

【スコープ】
- {{クラス全体 / 特定メソッド名}}
```

**カスタマイズのポイント:**

- `{{テスト対象ファイルパス}}`: テスト生成対象のクラスを指定（例: `src/main/java/com/example/service/OrderService.java`）
- `{{既存テストファイルパス}}`: チームの既存テストをスタイル参照元として指定。なければこの行を削除
- `{{クラス全体 / 特定メソッド名}}`: クラス全体を対象にするか、特定メソッドに絞るかを選択
- テストメソッド命名を英語にしたい場合は「`shouldReturnXxx_whenYyy`形式」に変更
- アサーションライブラリをHamcrestにしたい場合は「AssertJ」を「Hamcrest」に変更

### パターンB: Chat

特定メソッド1つだけのテストが欲しい場合に使用する。

```
以下のメソッドの単体テストをJUnit5 + Mockitoで書いてください。
正常系と異常系を含めてください。

対象メソッド: {{クラス名#メソッド名}}
```

## 人間が確認すべきこと

- [ ] テストの期待値（assert）がビジネスルールと一致しているか
- [ ] 境界値テストが含まれているか（0件、上限値、null等）
- [ ] Mockの設定が適切か（過剰なMock、検証漏れがないか）
- [ ] テストが実装の内部詳細に依存しすぎていないか（リファクタリングで壊れる脆いテストでないか）
- [ ] 実際にテストを実行して全件パスするか

## よくある落とし穴

1. **AI生成テストが「コードの写し」になりやすい**: AIは実装コードをそのまま逆転させてテストを作る傾向がある。仕様書やビジネスルールに基づく期待値になっているか、人間が必ず確認する。
2. **ビジネスルール固有の境界値を知らない**: AIは汎用的な境界値（null、空文字、0）は生成するが、業務固有の閾値（例: 税率切替の境界日、金額上限）は知らないため、人間が補完する必要がある。
3. **カバレッジ数値が高くてもテストの質が低い場合がある**: `assertNotNull` だけで通過するテストはカバレッジに貢献するが品質保証にならない。意味のあるアサーション（具体的な値の検証）になっているか確認する。
