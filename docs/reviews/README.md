# 多角的レビュー記録

## 概要

本ディレクトリには、プレイブックに対するPM・リーダー・開発者の3ロールによる多角的レビューの記録を蓄積する。

## 配置ルール

```
docs/reviews/
├── README.md                  # 本ファイル（配置ルール）
├── YYYY-MM-DD_NN/             # レビュー実施日 + 連番ごとのディレクトリ
│   ├── pm.md                  # PM視点レビュー
│   ├── leader.md              # リーダー視点レビュー
│   ├── dev.md                 # 開発者視点レビュー
│   └── summary.md             # ロール横断サマリー
└── ...
```

### 命名規則

| 項目 | ルール | 例 |
|------|--------|-----|
| ディレクトリ名 | `YYYY-MM-DD_NN`（実施日 + 2桁連番） | `2026-04-02_01/`, `2026-04-02_02/` |
| ロール別ファイル | `pm.md` / `leader.md` / `dev.md` | 固定名 |
| サマリーファイル | `summary.md` | 固定名 |

連番は同一日内で `01` から採番する。既存ディレクトリを確認し、最大の連番 +1 を使用すること。

### 各ファイルの構成（統一フォーマット）

```markdown
# {ロール名}視点レビュー — YYYY-MM-DD_NN

## レビュー対象
（対象ファイル・スコープを記載）

## 総合評価（5段階）

## 良い点（3〜5項目）

## 懸念・改善提案（優先度順に3〜5項目）

## 一言コメント
```

### summary.md の構成

```markdown
# ロール横断サマリー — YYYY-MM-DD_NN

## 3ロール共通の評価ポイント

## ロール間で意見が分かれた論点

## 優先的に対応すべき改善項目（統合・優先順位付け）
```

## 運用ガイド

### レビューの実行方法

Claude Code のスラッシュコマンドで実行する。

| コマンド | 用途 |
|---------|------|
| `/review-all` | 3ロール並列レビュー + 横断サマリー（推奨） |
| `/review-pm` | PM視点の単独レビュー |
| `/review-leader` | リーダー視点の単独レビュー |
| `/review-dev` | 開発者視点の単独レビュー |

引数で追加指示を渡せる（例: 特定ファイルに絞ったレビュー）。

### レビューの実施タイミング（目安）

- プレイブックの初版ドラフト完成時
- 大きな構成変更・UC追加時
- パイロット運用後のフィードバック反映後
- フェーズ移行の判断前

### 過去レビューの活用方法

- 新しいレビュー実施時に、前回の懸念・改善提案が対応されたか確認する
- summary.md の「優先的に対応すべき改善項目」を改善タスクのインプットにする
