# GitHub Copilot 3モード解説

## 1. Agent Mode

自律的にファイルの読み書き・ターミナル実行を行うモード。複数ファイルを跨ぐ作業に最適。

- プロジェクト全体のコンテキストを参照してコードを分析・生成できる
- ファイル作成・編集・削除、コマンド実行まで一貫して実行

### 使い方のコツ

- **既存パターンへの準拠を明示する** — 「既存の `XxxController` と同じ構成で作成して」のように指示
- **機能単位で分けて依頼する** — 1回のプロンプトで全部やらせず、機能ごとに区切る
- **生成結果は差分を確認してからコミットする**

### 注意点

- 大量ファイルを一度に変更すると確認が困難になる。スコープを絞ること
- ターミナル実行の提案は内容を確認してから承認する

---

## 2. Chat

チャットパネルでの対話形式。コードの理解・質問や、1ファイル内の小規模な作業に向く。

- プロジェクトコンテキストを参照して回答を生成
- 特定ファイルを指定して質問・依頼が可能

### 使い方のコツ

- 「このメソッドの処理フローを説明して」のような理解系タスクに強い
- コードレビュー観点での指摘依頼にも有効

### 注意点

- Chat の回答をそのままコピペしない。必ず内容を理解してから適用する
- 長い会話はコンテキストが薄れるため、テーマが変わったら新しいスレッドを開く

---

## 3. Inline Completion

エディタ上でリアルタイムに補完候補を表示するモード。定型的なコーディングに向く。

- コメントで意図を書く → 補完候補が表示される → Tab で確定
- getter/setter、定型パターン、繰り返し構造の記述が高速化

### 注意点

- 補完候補は必ず目視確認する。特に変数名・型の間違いに注意
- 複雑なロジックは Inline に頼らず、Agent Mode や Chat を使う

---

## Agent Mode と Chat の使い分け

| 判断基準 | Agent Mode | Chat |
|----------|------------|------|
| 変更ファイル数 | 複数ファイル | 1ファイル以内 |
| 作業内容 | コード生成・編集・実行 | 質問・理解・レビュー |
| 自律度 | 高い（ファイル操作まで実行） | 低い（回答のみ） |
| 典型例 | 新規クラス群の生成、リファクタ | 処理フローの確認、エラー調査 |
| リスク | 意図しない変更の混入 | コピペミス |

---

## 注意: IDE・ツールによる差異

本ガイドは **VS Code + GitHub Copilot Chat** を前提としています。以下の点に注意してください:

- **IntelliJ IDEA**: Agent Modeの挙動や利用可能な機能が異なる場合があります
- **他のAIツール**（Claude Code, Cursor等）: モードの概念が異なりますが、スキルファイルの内容はツール非依存で利用可能です
