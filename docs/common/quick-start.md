# クイックスタートガイド

## AI活用についての前提

**AIはスキルの代替ではなく、増幅ツールである。**

- AIの出力を読み解き、適切に判断・修正すること自体が高いスキルを要求される行為である
- レビューではコードの品質を見る。書いた手段（手書き／AI）は問わない
- AIの提案を「なぜそう書くのか」理解する過程で、むしろ学びが加速する
- 理解していないコードはAI生成であろうと手書きであろうとマージしない。この原則に変わりはない

## 事前準備（初回のみ・約15分）: プロジェクトへのAI設定ファイルの配置

本ガイドのスキルやエージェント定義を利用するには、まず `.github/` 配下の設定ファイルをプロジェクトに配置する。

1. 本リポジトリの `.github/` から以下をプロジェクトの `.github/` にコピーする
   - `copilot-instructions.md`（プロジェクト共通ルール）
   - `agents/`（エージェント定義・タスクメニュー）
   - `skills/`（タスク別スキル）
2. `copilot-instructions.md` をプロジェクトの技術スタック・規約に合わせてカスタマイズする（詳細は [CUSTOMIZATION.md](../../.github/CUSTOMIZATION.md) を参照）

> この手順を飛ばすと、スキル（`/cd-analyze-codebase` 等）やエージェント（`@implementer` 等）が利用できない。

> **「5分で試す」の所要時間について**: 以下の「5分で試す」は、上記の事前準備が完了している前提での所要時間です。事前準備（カスタマイズ含む）には約15分かかります。

## 5分で試す: 既存コードをAIに説明させる（事前準備完了後）

最もリスクが低く、効果を実感しやすいユースケース（[UC04: 既存コードベースの理解](../implementation/uc04-codebase-analysis.md)）を体験する。

### Step 1: Copilot Chatを開く

VS Codeで `Ctrl+Alt+I`（Mac: `Cmd+Alt+I`）を押してCopilot Chatパネルを開く。

### Step 2: 既存コードについて質問する

以下のプロンプトをCopilot Chatに入力する（ファイルパスは自分のプロジェクトに合わせて変更）。

```
#file:OrderService.java このクラスの処理フローを説明してください。
主要メソッドの役割と、他クラスとの依存関係を教えてください。
```

**ポイント:**
- `#file:ファイル名` で特定ファイルをコンテキストとして指定できる
- 質問するだけなので、コードは一切変更されない

### Step 3: 回答を検証する

AIの回答を読み、以下を確認する。

- [ ] 説明に登場するクラス名・メソッド名が実際に存在するか
- [ ] 処理フローの説明がコードの実態と合っているか

> AIが存在しないクラスやメソッドを説明に含めることがある（ハルシネーション）。回答は必ずコードと突き合わせること。

## うまくいかない時は

AIの出力が期待通りでないことは珍しくない。以下の対処法を試す。

1. **スコープを狭める** — 「クラス全体」ではなく「このメソッドだけ」に絞って質問し直す
2. **追加指示で軌道修正する** — 「〇〇の部分は既存の△△に合わせて修正して」と具体的に伝える
3. **モードを切り替える** — Agent Modeでうまくいかなければ、Chatで対話しながら手動編集に切り替える
4. **新しいスレッドで再開する** — 長い会話はコンテキストが薄れるため、テーマが変わったら新しいチャットを開く
