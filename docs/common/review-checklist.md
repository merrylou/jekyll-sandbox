# AI出力レビューチェックリスト

## 概要

AI生成物をレビューする際の共通チェックリスト。
工程固有の観点は各ユースケースの「人間が確認すべきこと」を参照すること。

---

## 全般チェック

- [ ] 要件・設計との整合性が取れているか
- [ ] AI生成物の内容を自分の言葉で説明できるか（理解していないコードはマージしない）
- [ ] ハルシネーション（存在しないクラス・メソッド・設定・API等）が含まれていないか
- [ ] 不要なコード・コメント・TODO が残っていないか
- [ ] ビルド・テストが通るか

## セキュリティチェック

- [ ] インジェクション脆弱性（SQL / Command / LDAP 等）がないか
- [ ] 認証・認可チェックが適切か
- [ ] 機密情報（パスワード・APIキー・トークン等）がハードコーディングされていないか
- [ ] ログに個人情報・機密情報が出力されていないか
- [ ] 入力バリデーションが適切か

## コード品質チェック

- [ ] プロジェクトの命名規則・コーディング規約に従っているか
- [ ] N+1問題・不要なループ等のパフォーマンス問題がないか
- [ ] 例外処理が適切か（握りつぶし、スタックトレースの露出等）
- [ ] 既存コードのパターン・スタイルと一貫しているか
- [ ] フレームワークのライフサイクルに起因する意図しない副作用がないか（アノテーション・フックの実行タイミング等）
- [ ] ORMの遅延/即時ロード設定がパフォーマンスに影響しないか
- [ ] DBスキーマが全環境（開発DB・本番DB等）で整合しているか（NULL制約・データ型の一致）

## ライセンス・知的財産

- [ ] OSSライセンスに抵触する可能性のあるコードが含まれていないか
- [ ] AIに機密情報（本番データ、顧客情報等）を入力していないか

---

## 使い方

1. PR提出前にこのチェックリストを確認する
2. 全項目OKでなくても構わない。NGの理由を把握した上でレビューに出すこと
3. レビュー依頼時に、確認済みの旨をPRコメントに記載する

---

## PRコメントでの活用例

PR提出時にセルフチェック結果を記載するテンプレート:

```markdown
## AI生成コードのセルフチェック
- [ ] 要件との整合性を確認済み
- [ ] AIのハルシネーション（実在しないAPI・ライブラリの使用等）がないことを確認済み
- [ ] ビルド・既存テストが通ることを確認済み
- [ ] セキュリティ観点（入力検証・認証認可・シークレット管理）を確認済み
- [ ] AIが追加した不要なコード（過剰な抽象化、未使用のimport等）を除去済み
```
