## チェック観点

### 共通観点

1. **バグ**: NullPointerException、境界値、例外処理漏れ等
2. **セキュリティリスク**: SQLインジェクション、XSS、認証認可の抜け等
3. **パフォーマンス問題**: N+1クエリ、不要なDB呼び出し、メモリリーク等
4. **コーディング規約違反**: プロジェクトの命名規則・スタイルとの不一致
5. **保守性**: 可読性、命名、責務の分離

### Spring Boot 固有観点

6. **フィールドインジェクション**: `@Autowired` によるフィールド注入を使っていないか → コンストラクタインジェクション + `private final` に変更すること
7. **エンティティ直接公開**: JPAエンティティをAPIレスポンスにそのまま返していないか → DTOに変換すること
8. **`@Transactional` の誤配置**: Controller層に `@Transactional` が付いていないか、またはServiceで必要なメソッドへの付与漏れがないか
9. **バリデーション漏れ**: Controllerのリクエストパラメータ・リクエストボディに `@Valid` / `@Validated` が付与されているか
10. **シークレットのハードコード**: パスワード・APIキー・接続文字列等がコードや `application.yml` に直書きされていないか → 環境変数参照 `${ENV_VAR}` を使用すること
11. **パスワードの安全性**: パスワードを平文で保存・比較していないか → BCrypt等の強いハッシュアルゴリズムを使用すること
12. **N+1問題**: ループ内でRepositoryを呼び出していないか、Lazyロードが意図せず大量クエリを発行していないか → `JOIN FETCH` またはDTOプロジェクションで解決すること
13. **DDL整合性**: 新規エンティティに対応するDDLが全スキーマファイル（h2/mysql/postgres等）に存在するか
14. **テスト網羅**: 新規追加したController/Serviceに対応するテストファイルが存在するか
15. **`@ModelAttribute` / フィルターの重複実行**: 同一リクエスト内で同一データを複数回取得していないか（`@ModelAttribute`メソッド間の依存関係に注意）

## 出力形式

| 深刻度 | ファイル | 行 | 問題内容 | 改善案 |

深刻度はCritical / High / Medium / Lowで分類すること。

深刻度の判定基準:
- **Critical**: 本番障害・データ損失・セキュリティインシデントに直結する
- **High**: 機能が正しく動作しない、または修正しないと後続作業に影響する
- **Medium**: 動作するが設計上好ましくない、将来の保守性に影響する
- **Low**: コードスタイル・可読性・命名の改善

## 必要な情報（未指定なら質問すること。対話できない場合は妥当な仮定を置き、仮定した内容を出力に明記すること）

- 変更ファイルの一覧（`git diff --name-only` の結果でも可）
- チーム固有のコーディング規約ファイル（あれば）
