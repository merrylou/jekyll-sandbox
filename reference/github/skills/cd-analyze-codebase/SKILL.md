## 出力構成

### 1. システム全体概要

このシステムが何をするものか、対象ユーザー・主要機能・業務上の位置づけを3〜5行で説明してください。
コードから読み取れる事実（エンドポイント名、ドメインクラス名、設定ファイル等）を根拠にすること。


### 3. アーキテクチャ概要

レイヤー構成と主要パッケージの関係を **Mermaid の `graph TD`** で図示してください。
各ノードには実際のパッケージ名またはクラス名を使い、レイヤー間の依存方向を矢印で示すこと。

```mermaid
graph TD
  （ここに実際のコードから読み取った構成を記載）
```

図の後に、各レイヤーの役割を1〜2行で補足してください。


### 5. データの流れ

入力データがどのように変換・加工・永続化されるかを **Mermaid の `flowchart LR`** で図示してください。

```mermaid
flowchart LR
  （ここにDTO/Entity/DB等のデータ変換の流れを記載）
```

図の後に、主要なDTOやEntityクラスの役割を1〜2行で補足してください。


### 7. 外部依存

DB、外部API、メッセージング等の依存先を一覧化してください。
設定ファイル（application.yml 等）から読み取れる情報を根拠にすること。


## 注意

- クラス名・メソッド名・パッケージ名は実際にコードベースに存在するもののみを使うこと
- 推測や一般論ではなく、実際のコードに基づいて回答すること
- Mermaid の記法エラーに注意し、正しく描画できる構文で記述すること

## ファイル出力

分析結果は以下のルールに従ってファイルに保存してください。

### 出力先

```
docs/analysis/
└── YYYY-MM-DD_NN/
    └── onboarding.md
```

- `YYYY-MM-DD` は実行日の日付
- `NN` は同一日内の2桁連番（既存ディレクトリを確認し、最大の連番 +1 を採番。初回は `01`）
- ディレクトリが存在しない場合は作成すること

### ファイルの冒頭

```markdown
# コードベース概要 — YYYY-MM-DD_NN

## 分析対象
（対象パッケージ・モジュール・機能領域を記載）

## 分析実施者メモ
（未指定の場合は空欄のまま残す）
```

その後、「出力構成」の1〜7をそのまま続けて記載してください。
