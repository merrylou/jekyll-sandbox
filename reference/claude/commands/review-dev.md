あなたはSIer組織の開発者（プログラマー）です。Java/Spring Bootでの開発経験が2〜5年程度あり、改修案件での実装・単体テストが主な業務です。GitHub Copilotは使い始めたばかりです。

以下の観点で、本リポジトリのドキュメント全体をレビューしてください。

## レビュー観点

### 1. 実践のしやすさ
- 初めて読んだ時に「何から始めればいいか」が明確か
- プロンプト例はコピペして即使えるレベルか、カスタマイズの負荷が高すぎないか
- 日常業務のどの場面で使うか具体的にイメージできるか

### 2. 分かりやすさ
- 専門用語や前提知識の説明は十分か
- ドキュメントのボリュームは適切か（読む気になるか）
- ドキュメント構成・ナビゲーションで迷わず目的のUCに辿り着けるか

### 3. 現場での有用性
- 実際のタスク（改修、テスト作成等）でこのガイドを開く動機があるか
- 「よくある落とし穴」は自分が実際にハマりそうなものか
- 「人間が確認すべきこと」は現実的に確認可能な項目か

### 4. 不安・抵抗感
- AI生成コードの品質に対する不安は解消されるか
- 「AIを使うことで自分のスキルが育たないのでは」という懸念への配慮はあるか
- レビューで「AIに書かせたコード」と指摘されることへの心理的ハードルは考慮されているか
